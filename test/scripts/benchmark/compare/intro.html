Introduction

The following table shows benchmarking results of various tests for the available prover engines in GeoGebra:

  • Columns "Classic5-*" refers to the Java version of GeoGebra (also known as "desktop" version). If indicated, here * can be "m" for Mathematica and "t" for Tarski as underlying backends.
  • Column "Classic6" refers to the HTML/JavaScript version of GeoGebra (also known as "web" version).

Explanation of the used colors:

  • Green means that the test returns a positive answer. Intensity of green means speed (the lighter the slower). Numbers are in milliseconds.
  • Pink means that GeoGebra returns the wrong answer.
  • Yellow means the output is not conclusive, thus using this method GeoGebra shows "undefined", i.e. there is no error here.
  • The Result column provides information about the result, that is, the output of the comparison.

Based on the output results the number of positive tests is shown at the bottom of the table.
